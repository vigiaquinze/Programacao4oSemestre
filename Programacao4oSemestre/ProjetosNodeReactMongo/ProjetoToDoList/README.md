<center><h1>Desenvolvimento de Aplicação To-Do List</h1></center>

## Objetivo
Desenvolver uma aplicação web de To-Do List que permita aos usuários adicionar, editar, excluir e visualizar suas tarefas de forma intuitiva e segura. A aplicação deve ser construída com React para o front-end, Node.js para o back-end, MongoDB para o banco de dados e JWT para autenticação.

## Objetivos SMART
- ### Específico:
    Construir uma aplicação de To-Do List onde os usuários podem se autenticar, gerenciar suas tarefas e visualizar uma lista de tarefas pendentes e concluídas.
- ### Mensurável:
    Implementar funcionalidades de autenticação, CRUD (criar, ler, atualizar, deletar) de tarefas e gerenciamento de usuários, com 90% de cobertura de testes automatizados e performance adequada.
- ### Atingível:
    Utilizar tecnologias já conhecidas e práticas recomendadas, com uma equipe de desenvolvimento pequena e foco na implementação ágil e contínua.
- ### Relevante:
    Atender à necessidade da Escola SENAI de uma ferramenta eficaz para a organização de tarefas, garantindo segurança e uma experiência de usuário otimizada.
- ### Temporal:
    Concluir o desenvolvimento e o lançamento da aplicação em 12 semanas, com marcos intermediários a cada 2 semanas.

## Cronograma
- ### Semana 1-2: Planejamento e design
    - Definição dos requisitos do projeto.
    - Design do wireframe e arquitetura da aplicação.

- ### Semana 3-4: Configuração do ambiente de desenvolvimento
    - Configuração dos ambientes de desenvolvimento e ferramentas necessárias.
    - Preparação do repositório e estrutura inicial do projeto.

- ### Semana 5-6: Desenvolvimento do back-end
    - Implementação da API em Node.js.
    - Configuração do MongoDB para armazenamento de dados.
    - Implementação da autenticação com JWT.

- ### Semana 7-8: Desenvolvimento do Front-End
    - Implementação da interface de usuário com React.
    - Integração do front-end com a API do back-end.

- ### Semana 9: Integração e Testes
    - Integração do front-end com o back-end.
    - Início dos testes unitários e de integração.

- ### Semana 10: Ajustes e Melhorias
    - Correção de bugs identificados nos testes.
    - Ajustes baseados no feedback recebido e melhorias de performance.

- ### Semana 11: Documentação e Treinamento
    - Criação de documentação técnica detalhada.
    - Preparação e realização de treinamento para os usuários finais.

- ### Semana 12: Lançamento e Avaliação
    - Lançamento da aplicação para o público.
    - Coleta de feedback dos usuários e avaliação pós-lançamento para possíveis melhorias.
